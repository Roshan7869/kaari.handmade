## Deploy in 5 Minutes

### Step 1: Prepare Credentials (2 min)

Go to https://supabase.com/dashboard and get:
- **NEXT_PUBLIC_SUPABASE_URL**
- **NEXT_PUBLIC_SUPABASE_ANON_KEY**
- **SUPABASE_SERVICE_KEY**

Copy these - you'll need them in a moment.

### Step 2: Go to Vercel (1 min)

Visit https://vercel.com and sign in with GitHub

### Step 3: Deploy (2 min)

1. Click **"Add New"** → **"Project"**
2. Select your **"kaari-handmade"** repository
3. Click **"Import"**
4. Click **"Environment Variables"**
5. Paste your Supabase credentials
6. Click **"Deploy"**

**That's it! Your site is now live!** 🎉

## Common Questions

**Q: Can people see my site without signing up?**
A: Yes! Your homepage and products are public. Only checkout requires login.

**Q: How much does it cost?**
A: Vercel free tier is included. You pay when you exceed free limits (~millions of requests).

**Q: Can I use my own domain?**
A: Yes! Buy a domain and connect it in Vercel Settings → Domains.

**Q: What if something breaks?**
A: Check Vercel build logs or see troubleshooting in **[VERCEL_DEPLOYMENT_GUIDE.md](./VERCEL_DEPLOYMENT_GUIDE.md)**.

**Q: How do I update my site?**
A: Push code changes to GitHub and Vercel automatically deploys.

**Q: Can I run locally first?**
A: Yes! Run `npm install` then `npm run dev` to test locally on http://localhost:3000.

## Quick Troubleshooting

### "Build failed"
→ Check build logs in Vercel dashboard. Usually a missing environment variable.

### "Can't see products"
→ Make sure Supabase database is set up with products table.

### "Login doesn't work"
→ Verify NEXT_PUBLIC_SUPABASE_URL and keys are correct.

### "Site shows 404"
→ Check that all routes exist in `/app` directory.

**For more help:** See troubleshooting in **[VERCEL_DEPLOYMENT_GUIDE.md](./VERCEL_DEPLOYMENT_GUIDE.md)**
